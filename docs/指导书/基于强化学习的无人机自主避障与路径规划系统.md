适用课程：《人工智能实践》  
适用对象：具有 Python、ROS 基础和基本机器学习知识的本科学生

## 1. 项目概述

### 1.1 项目背景

本项目面向无人机在动态障碍物环境中的自主导航问题，要求学生在已有 ROS 无人机仿真平台上，将传统控制方法与强化学习方法结合，实现“能稳定飞行、能感知障碍、能在线决策、能量化评估”的完整实验闭环。

现有工程已经包含以下基础能力：

- Gazebo 仿真环境和 ROS 通信基础。
- MAVROS 与 PX4 SITL 的无人机控制接口。
- `lqr_trajectory_tracker` 包，提供基于 LQR 的轨迹跟踪控制器。
- Velodyne 激光雷达仿真相关包，可作为环境感知数据来源。
- `src/lqr_trajectory_tracker/worlds/obstacles_only.world` 静态障碍物场景，可作为场景扩展起点。

本实践不建议学生直接从强化学习网络输出底层电机控制量。更合理的工程分层是：强化学习负责高层避障决策或局部航点选择，LQR 负责低层轨迹跟踪，PX4 负责飞控闭环。这种分层更稳定，也更适合课程项目复现。

### 1.2 项目目标

本项目的核心目标如下：

1. 在现有 LQR 轨迹跟踪控制器基础上，引入 DQN 或 PPO，实现局部动态避障。
2. 构建不同复杂度的障碍物仿真场景，覆盖稀疏、密集、静态、动态等情况。
3. 实现 A*、RRT 等传统路径规划基线，并与强化学习方法进行对比。
4. 形成可复现实验流程，包括场景配置、训练参数、日志记录、评价指标和结果分析。

### 1.4 成功标准

完成项目时，至少应满足以下标准：

- 无障碍场景下，LQR 能稳定跟踪直线、圆形或固定水平 8 字形轨迹。
- 静态障碍物场景下，强化学习策略能在多数测试回合中绕开障碍物到达目标。
- 动态障碍物场景下，策略能根据感知变化调整局部航点或动作。
- 实验报告中包含 DQN/PPO 与 A*/RRT 的对比结果。
- 所有实验结果能通过固定随机种子和配置文件复现。

## 2. 技术方案设计

### 2.1 总体架构

推荐采用“全局目标 + 局部避障 + 低层跟踪”的混合架构：

```text
全局目标点
    |
    v
全局规划器 A*/RRT 或预设参考轨迹
    |
    v
局部强化学习避障器
    |
    v
局部目标点 / 局部速度方向
    |
    v
LQR 轨迹跟踪控制器
    |
    v
MAVROS / PX4 / Gazebo
```

各模块职责应保持清晰：

| 模块 | 输入 | 输出 | 职责 |
|------|------|------|------|
| 环境感知模块 | 激光雷达点云、无人机位姿 | 障碍物特征、占据栅格 | 将原始传感器数据转换为学习算法可用状态 |
| 强化学习决策模块 | 状态向量、目标方向、历史动作 | 离散动作或连续局部目标 | 决定避障方向和短期机动策略 |
| LQR 跟踪模块 | 参考轨迹点、当前位姿 | MAVROS 位置设定点 | 稳定跟踪局部目标或轨迹 |
| 场景管理模块 | 场景配置、随机种子 | Gazebo world、障碍物状态 | 生成和管理训练、验证、测试场景 |
| 实验评估模块 | 飞行日志、碰撞信息、规划日志 | 指标表格、图表 | 评价算法性能和鲁棒性 |

### 2.2 LQR 与强化学习的接口设计

项目已有 `lqr_trajectory_tracker` 节点默认订阅：

- `/mavros/local_position/pose`
- `/mavros/state`

默认发布：

- `/mavros/setpoint_position/local`
- `/lqr_tracker/accel_cmd`
- `/lqr_tracker/ref_trajectory`
- `/lqr_tracker/position_error`
- `/lqr_tracker/velocity_error`
- `/lqr_tracker/total_error`

为了接入强化学习，建议保留 LQR 的低层控制职责，在其前面新增一个局部规划接口。推荐两种实现方式：

方式一：RL 输出局部目标点。

```text
rl_planner_node 发布 /rl_planner/local_goal geometry_msgs/PoseStamped
lqr_tracker_node 订阅 /rl_planner/local_goal 并将其作为短期参考点
```

方式二：RL 输出离散动作，由桥接节点转换成局部目标点。

```text
动作 0: 前进
动作 1: 左移
动作 2: 右移
动作 3: 上升
动作 4: 下降
动作 5: 减速或悬停
```

对本科实践，优先推荐方式二。离散动作更容易用 DQN 实现，也更容易调试和解释。

### 2.3 强化学习问题建模

#### 2.3.1 状态空间

推荐状态向量包含以下信息：

| 状态项 | 说明 | 示例维度 |
|--------|------|----------|
| 激光雷达分区最小距离 | 将前方或水平面划分为若干扇区，记录每个扇区最近障碍物距离 | 12 或 24 |
| 目标相对方向 | 当前无人机指向目标点的单位向量 | 3 |
| 目标相对距离 | 当前点到目标点距离，归一化到 `[0, 1]` | 1 |
| 当前速度 | 来自里程计或位姿差分 | 3 |
| 上一步动作 | 帮助减少策略抖动 | 1 或动作数 one-hot |

一个简化状态可以定义为：

```text
state = [
    lidar_min_dists[0:12],
    target_dir_x, target_dir_y, target_dir_z,
    target_dist_norm,
    vel_x, vel_y, vel_z,
    last_action
]
```

#### 2.3.2 动作空间

DQN 推荐使用离散动作：

| 动作编号 | 动作含义 | 局部目标偏移示例 |
|----------|----------|------------------|
| 0 | 前进 | `(step, 0, 0)` |
| 1 | 左移 | `(0, step, 0)` |
| 2 | 右移 | `(0, -step, 0)` |
| 3 | 上升 | `(0, 0, step_z)` |
| 4 | 下降 | `(0, 0, -step_z)` |
| 5 | 悬停/减速 | `(0, 0, 0)` |

PPO 可使用连续动作：

```text
action = [delta_x, delta_y, delta_z]
```

连续动作需要增加限幅、平滑和安全检查，否则训练早期容易产生剧烈机动。

#### 2.3.3 奖励函数

奖励函数应同时鼓励到达目标、远离障碍、减少绕路和保持平滑。推荐从简单版本开始：

```text
r = w_goal * progress
  + w_success * success_bonus
  - w_collision * collision_penalty
  - w_near * near_obstacle_penalty
  - w_step * step_penalty
  - w_smooth * action_change_penalty
```

各项含义：

| 奖励项 | 建议值 | 说明 |
|--------|--------|------|
| `progress` | `prev_dist - curr_dist` | 靠近目标给正奖励，远离目标给负奖励 |
| `success_bonus` | `+100` | 到达目标后给一次性奖励 |
| `collision_penalty` | `-100` | 碰撞或距离障碍过近时终止回合 |
| `near_obstacle_penalty` | `-1` 到 `-10` | 障碍物距离小于安全阈值时触发 |
| `step_penalty` | `-0.01` 到 `-0.1` | 鼓励更短路径 |
| `action_change_penalty` | `-0.05` | 减少左右横跳和轨迹抖动 |

不要一开始设计过于复杂的奖励。建议先验证“无障碍能到达目标”，再加入“静态障碍绕行”，最后加入“动态障碍”。

### 2.4 DQN 与 PPO 选择建议

| 算法 | 适用情况 | 优点 | 难点 |
|------|----------|------|------|
| DQN | 离散动作、低维状态 | 实现简单，适合教学 | 动作不够细腻，连续控制能力弱 |
| PPO | 连续动作、策略优化 | 稳定性较好，适合局部目标输出 | 训练成本更高，参数更多 |

课程项目建议路线：

1. 第一阶段使用 DQN 完成离散动作避障。
2. 第二阶段可选做 PPO，将动作改为连续局部目标偏移。
3. 对比实验中，DQN/PPO 至少保留一个强化学习方法与 A*/RRT 对比。

### 3.4 数据预处理

激光雷达点云直接输入神经网络成本较高。课程项目建议使用扇区最小距离特征：

1. 将点云转换到无人机机体系或局部地图坐标系。
2. 只保留安全高度附近或无人机周围固定半径内的点。
3. 将水平角度划分为 `N` 个扇区，例如 12、24 或 36。
4. 对每个扇区取最近障碍物距离。
5. 对距离进行裁剪和归一化，例如 `d_norm = min(d, max_range) / max_range`。

伪代码：

```python
def lidar_to_sectors(points, num_sectors=24, max_range=10.0):
    sectors = [max_range] * num_sectors
    for x, y, z in points:
        if abs(z) > 1.5:
            continue
        dist = (x * x + y * y) ** 0.5
        if dist < 0.2 or dist > max_range:
            continue
        angle = math.atan2(y, x)
        idx = int((angle + math.pi) / (2 * math.pi) * num_sectors)
        idx = min(max(idx, 0), num_sectors - 1)
        sectors[idx] = min(sectors[idx], dist)
    return [d / max_range for d in sectors]
```

### 4.2 环境封装

强化学习训练需要一个类似 Gym 的环境类，将 ROS/Gazebo 交互封装为 `reset()` 和 `step(action)`。

核心接口：

```python
class DroneAvoidanceEnv:
    def __init__(self, config):
        self.config = config
        self.max_steps = config["max_steps"]
        self.safe_distance = config["safe_distance"]

    def reset(self):
        self.reset_world()
        self.set_drone_pose(self.config["start"])
        self.goal = self.sample_goal()
        self.step_count = 0
        return self.get_observation()

    def step(self, action):
        local_goal = self.action_to_local_goal(action)
        self.publish_local_goal(local_goal)
        self.wait_control_dt()

        obs = self.get_observation()
        reward = self.compute_reward(obs)
        done = self.is_done(obs)
        info = self.collect_info()
        self.step_count += 1
        return obs, reward, done, info
```

注意事项：

- `reset()` 不应只重置神经网络状态，还要重置无人机位置、目标点和障碍物状态。
- `step()` 中的控制周期建议从 `0.1s` 到 `0.2s` 开始，不要与 LQR 的 50Hz 低层控制周期混在一起。
- 每个回合设置最大步数，避免无人机卡在障碍物前无限训练。

### 4.3 DQN 实现建议

DQN 适合离散动作。基本组件包括 Q 网络、目标网络、经验回放和 epsilon-greedy 探索。

网络结构可从小模型开始：

```python
class QNetwork(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim),
        )

    def forward(self, x):
        return self.net(x)
```

训练流程：

1. 初始化在线 Q 网络和目标 Q 网络。
2. 用 epsilon-greedy 选择动作。
3. 执行动作，保存 `(s, a, r, s_next, done)` 到 replay buffer。
4. 从 buffer 采样 batch。
5. 使用 Bellman 目标更新在线网络。
6. 每隔固定步数同步目标网络。

DQN 损失：

```text
target = reward + gamma * max_a Q_target(next_state, a) * (1 - done)
loss = MSE(Q_online(state, action), target)
```

推荐超参数初值：

| 参数 | 建议值 |
|------|--------|
| 学习率 | `1e-4` 到 `5e-4` |
| 折扣因子 `gamma` | `0.98` 或 `0.99` |
| replay buffer | `50000` |
| batch size | `64` 或 `128` |
| epsilon 初值 | `1.0` |
| epsilon 终值 | `0.05` |
| 目标网络同步周期 | `500` 到 `2000` 步 |

### 4.4 PPO 实现建议

PPO 适合连续动作或较复杂策略。推荐使用 actor-critic 结构：

```text
Actor: state -> action distribution
Critic: state -> state value
```

连续动作可定义为局部目标偏移：

```text
delta_x in [-1.0, 1.0]
delta_y in [-1.0, 1.0]
delta_z in [-0.5, 0.5]
```

PPO 训练时需要收集一批 rollout，然后多轮优化 clipped objective：

```text
L_clip = min(r_t(theta) * A_t, clip(r_t(theta), 1-eps, 1+eps) * A_t)
```

推荐超参数初值：

| 参数 | 建议值 |
|------|--------|
| 学习率 | `3e-4` |
| rollout steps | `1024` 或 `2048` |
| batch size | `64` |
| PPO epoch | `5` 到 `10` |
| clip ratio | `0.2` |
| entropy coef | `0.01` |
| value loss coef | `0.5` |

如果课程时间有限，可以使用 Stable-Baselines3 训练 PPO，但要求学生清楚说明状态、动作、奖励和终止条件，而不是只调用库函数。

### 4.5 RL 到 LQR 的桥接节点

桥接节点负责把动作转换为局部目标点，并加入安全限幅。

动作转换示例：

```python
def action_to_offset(action, step_xy=1.0, step_z=0.5):
    table = {
        0: (step_xy, 0.0, 0.0),
        1: (0.0, step_xy, 0.0),
        2: (0.0, -step_xy, 0.0),
        3: (0.0, 0.0, step_z),
        4: (0.0, 0.0, -step_z),
        5: (0.0, 0.0, 0.0),
    }
    return table[action]
```

局部目标发布示例：

```python
def publish_local_goal(pub, current_pose, offset):
    msg = PoseStamped()
    msg.header.stamp = rospy.Time.now()
    msg.header.frame_id = "map"
    msg.pose.position.x = current_pose.pose.position.x + offset[0]
    msg.pose.position.y = current_pose.pose.position.y + offset[1]
    msg.pose.position.z = max(1.0, current_pose.pose.position.z + offset[2])
    msg.pose.orientation = current_pose.pose.orientation
    pub.publish(msg)
```

工程上必须增加以下保护：

- 局部目标点不能低于最小飞行高度。
- 局部目标点不能超过场景边界。
- 若最近障碍物距离小于紧急阈值，优先发布悬停或后退动作。
- 动作变化过快时应做限幅或平滑。

### 4.6 传统路径规划基线

A* 和 RRT 不需要直接接管低层飞控，可输出全局或局部航点，再交给 LQR 跟踪。

A* 实现建议：

- 将环境离散为 2D 或 3D 占据栅格。
- 静态障碍物膨胀安全半径，例如 `0.5m` 到 `1.0m`。
- 使用欧氏距离或曼哈顿距离作为启发函数。
- 输出栅格路径后进行路径平滑。

RRT 实现建议：

- 在连续空间中随机采样。
- 每次从树中最近节点向采样点扩展固定步长。
- 碰撞检测使用线段与障碍物膨胀体的距离。
- 找到路径后进行 shortcut 平滑。

对比时要保持公平：

- 起点、目标点和障碍物配置相同。
- 最大规划时间相同。
- 碰撞安全半径相同。
- LQR 跟踪器参数相同。

### 5.2 分阶段训练策略

不要直接在复杂动态场景中训练。建议按以下阶段推进：

| 阶段 | 场景 | 目标 | 验证标准 |
|------|------|------|----------|
| Stage 0 | 无障碍 | 控制链路能跑通 | 能到达目标，无碰撞 |
| Stage 1 | 单个静态障碍 | 学会绕行 | 成功率超过 80% |
| Stage 2 | 稀疏静态障碍 | 学会多障碍避让 | 成功率超过 70% |
| Stage 3 | 密集静态障碍 | 提升鲁棒性 | 碰撞率下降，路径不过度绕行 |
| Stage 4 | 动态障碍 | 在线避障 | 遇到移动障碍能减速、绕行或悬停 |

每个阶段都应固定训练场景和测试场景。训练场景用于学习，测试场景只用于最终评估。

### 5.3 DQN 训练伪代码

```python
env = DroneAvoidanceEnv(config)
agent = DQNAgent(state_dim, action_dim)

for episode in range(num_episodes):
    state = env.reset()
    episode_reward = 0

    for t in range(max_steps):
        action = agent.select_action(state)
        next_state, reward, done, info = env.step(action)
        agent.replay_buffer.add(state, action, reward, next_state, done)
        agent.update()

        state = next_state
        episode_reward += reward

        if done:
            break

    agent.update_epsilon()
    logger.log_episode(episode, episode_reward, info)
    if episode % save_interval == 0:
        agent.save(checkpoint_path)
```

### 5.4 验证流程

训练完成后，必须关闭探索噪声或将 epsilon 设置为最小值，再进行验证：

```text
for each algorithm in [DQN, PPO, A*, RRT]:
    for each scenario in test_scenarios:
        for seed in [0, 1, 2, 3, 4]:
            reset scenario
            run one episode
            record metrics
```

验证时禁止继续训练网络，否则测试结果不具备可比性。

### 5.5 常见训练问题

| 问题 | 可能原因 | 处理方法 |
|------|----------|----------|
| 奖励一直很低 | 奖励过稀疏，成功奖励太难拿到 | 加入靠近目标的 progress 奖励 |
| 无人机原地不动 | 悬停惩罚不够或动作转换失败 | 检查 `/rl_planner/local_goal` 是否发布 |
| 频繁撞障碍物 | 安全距离过小，训练从困难场景开始 | 降低场景难度，增加 near obstacle penalty |
| 左右抖动 | 动作切换惩罚缺失 | 增加 action change penalty |
| Gazebo 训练很慢 | 仿真渲染开销大 | 使用 headless 或降低传感器频率 |
| LQR 跟踪误差大 | 局部目标跳变过大 | 减小动作步长或增加目标平滑 |

## 7. 评估与分析方法

### 7.1 评价指标定义

#### 7.1.1 路径长度优化率

路径长度：

```text
L = sum(||p_t - p_{t-1}||)
```

路径长度优化率可相对于基线算法定义：

```text
optimization_rate = (L_baseline - L_method) / L_baseline * 100%
```

若强化学习路径更短，该指标为正；若更长，则为负。

#### 7.1.2 避障成功率

```text
success_rate = successful_episodes / total_episodes * 100%
```

成功条件建议同时满足：

- 到达目标距离小于 `goal_tolerance`。
- 回合中没有碰撞。
- 没有超出场景边界。
- 未超过最大步数。

#### 7.1.3 碰撞率

```text
collision_rate = collision_episodes / total_episodes * 100%
```

碰撞可以通过 Gazebo contact sensor 记录，也可以先用保守距离阈值近似：

```text
min_obstacle_distance < safety_radius
```

#### 7.1.4 计算效率

规划时间包括：

- 强化学习策略推理时间。
- A*/RRT 单次规划时间。
- 感知特征生成时间。

建议记录平均值、最大值和 95 分位数：

```text
mean_time_ms
max_time_ms
p95_time_ms
```

#### 7.1.5 轨迹平滑度

可使用加速度变化或转角变化度量：

```text
smoothness = sum(||a_t - a_{t-1}||)
```

如果没有加速度数据，可用路径点夹角近似：

```text
theta_t = angle(p_t - p_{t-1}, p_{t+1} - p_t)
smoothness = sum(abs(theta_t))
```

### 7.2 指标汇总表

实验结果建议按以下格式汇总：

| 算法 | 场景 | 成功率 | 碰撞率 | 平均路径长度 | 平均规划时间 | 平滑度 |
|------|------|--------|--------|--------------|--------------|--------|
| A* | S1 |  |  |  |  |  |
| RRT | S1 |  |  |  |  |  |
| DQN | S1 |  |  |  |  |  |
| PPO | S1 |  |  |  |  |  |

### 7.3 可视化分析

建议输出以下图表：

- 训练奖励曲线。
- 成功率随训练步数变化曲线。
- 不同算法路径对比图。
- 障碍物场景俯视图。
- 最小障碍物距离随时间变化曲线。
- LQR 位置跟踪误差曲线。
- 动作分布柱状图。

轨迹分析示例：

```python
def compute_path_length(positions):
    length = 0.0
    for i in range(1, len(positions)):
        length += np.linalg.norm(positions[i] - positions[i - 1])
    return length

def compute_smoothness(positions):
    if len(positions) < 3:
        return 0.0
    total = 0.0
    for i in range(1, len(positions) - 1):
        v1 = positions[i] - positions[i - 1]
        v2 = positions[i + 1] - positions[i]
        denom = np.linalg.norm(v1) * np.linalg.norm(v2) + 1e-8
        cos_theta = np.clip(np.dot(v1, v2) / denom, -1.0, 1.0)
        total += abs(np.arccos(cos_theta))
    return total
```

### 7.4 结果解释要求

报告中不能只贴表格，应解释以下问题：

- 为什么某个算法路径更短但碰撞率更高。
- 为什么强化学习在训练场景表现好，但测试场景下降明显。
- 动态障碍物对 A*/RRT 和 DQN/PPO 的影响有何不同。
- LQR 跟踪误差是否影响最终避障结果。
- 策略失败的典型案例是什么，是否与感知盲区或奖励设计有关。

### 8.2 安全策略

强化学习策略不能被完全信任，部署时必须增加规则保护层：

| 风险 | 保护策略 |
|------|----------|
| 距离障碍过近 | 强制悬停、后退或上升 |
| 局部目标超边界 | 裁剪到合法区域 |
| 高度过低 | 强制提升到最小高度 |
| 策略输出频繁跳变 | 限制局部目标变化率 |
| 传感器数据中断 | 保持悬停并报警 |

安全过滤器示例：

```python
def safety_filter(local_goal, current_pose, min_dist, bounds):
    if min_dist < 0.5:
        local_goal.x = current_pose.x
        local_goal.y = current_pose.y
        local_goal.z = current_pose.z + 0.5

    local_goal.x = np.clip(local_goal.x, bounds["x"][0], bounds["x"][1])
    local_goal.y = np.clip(local_goal.y, bounds["y"][0], bounds["y"][1])
    local_goal.z = np.clip(local_goal.z, bounds["z"][0], bounds["z"][1])
    return local_goal
```

### 8.3 模型导出与推理

训练完成后，推理节点只加载模型并执行前向传播，不应包含训练逻辑。

PyTorch 推理建议：

```python
model = QNetwork(state_dim, action_dim)
model.load_state_dict(torch.load(model_path, map_location="cpu"))
model.eval()

with torch.no_grad():
    q_values = model(torch.tensor(obs, dtype=torch.float32).unsqueeze(0))
    action = int(torch.argmax(q_values, dim=1).item())
```

如果使用 GPU 训练，部署时也要验证 CPU 推理速度。避障策略的单次推理时间建议低于 `20ms`。

### 8.4 从仿真到真实系统的注意事项

本课程项目以仿真为主。如果扩展到真实无人机，必须额外考虑：

- 传感器噪声和延迟。
- 定位漂移。
- 风扰和动力学模型差异。
- 飞控 failsafe 设置。

- 法规和飞行安全。

仿真中训练出的策略不能直接用于真实飞行，至少需要进行域随机化、硬件在环测试和安全飞行边界验证。
